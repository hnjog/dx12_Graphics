# Review Rules

## 목적

이 문서는 `dx12Engine` 프로젝트의 코드 리뷰 기준을 정의합니다.
AI Agent와 사람이 같은 우선순위와 판단 기준으로 리뷰할 수 있도록 핵심 원칙을 정리합니다.

## 리뷰 우선순위

코드 리뷰는 아래 순서로 우선 검토합니다.

1. 크래시, 메모리 손상, 핸들 누수 가능성
2. DirectX 12 리소스 수명 및 동기화 문제
3. 잘못된 상태 전이, API 오용, 초기화 누락
4. 기능 회귀 및 테스트 누락
5. 성능 저하 가능성
6. 유지보수성 문제
7. 스타일 및 표현 개선

## 심각도 기준

- `Blocker`: 머지하면 안 되는 수준의 문제
- `Major`: 기능 안정성, 리소스 관리, 회귀 가능성에 큰 영향이 있는 문제
- `Minor`: 수정이 권장되지만 즉시 치명적이지 않은 문제
- `Suggestion`: 품질 개선 제안

## 공통 리뷰 체크 항목

- 리소스 생성, 소유권, 해제 경로가 명확한가
- Command List와 Command Queue 사용 순서가 안전한가
- 상태 전이와 동기화가 누락되지 않았는가
- `Debug`와 `Release`, `x64` 기준에서 동작 가정이 타당한가
- 변경 내용에 맞는 테스트 또는 확인 절차가 포함되었는가
- 새 코드가 기존 구조를 불필요하게 복잡하게 만들지 않는가

## AI Review Rules

- 스타일보다 동작 안정성과 회귀 가능성을 우선 지적합니다.
- 한 코멘트에는 하나의 핵심 이슈만 담습니다.
- 가능하면 문제 위치, 위험, 예상 영향 범위를 함께 설명합니다.
- 확실하지 않은 내용은 단정하지 말고 가정임을 명시합니다.
- 단순 취향 차이는 강한 표현으로 지적하지 않습니다.

## Human Review Rules

- 리뷰는 수정 요청과 제안을 구분해서 남깁니다.
- 코드 작성 의도를 먼저 이해한 뒤 구조적 문제를 판단합니다.
- 재현 가능한 문제는 확인 방법을 함께 남깁니다.
- 대안이 있다면 방향만이 아니라 이유도 함께 설명합니다.

## Pull Request Expectations

PR 설명에는 최소한 아래 내용을 포함합니다.

- 변경 목적
- 주요 변경 사항
- 빌드 또는 실행 확인 여부
- 테스트 또는 수동 검증 방법
- 알려진 제한 사항 또는 남은 작업
